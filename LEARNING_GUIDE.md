# 📚 Linux 네트워킹 깊이 있는 학습 가이드

## 🎯 이 가이드의 목적

Linux 커널의 네트워킹부터 Kubernetes, eBPF, Cilium까지 **완벽히 이해**하기 위한 학습 경로를 제시합니다.

---

## 📖 학습 방식

### ✅ 추천하는 방법

```
1️⃣ 각 챕터의 README 읽기 (5분)
   → 학습 목표와 선수 지식 확인

2️⃣ Theory.md 정독 (이론)
   → "왜"를 이해하기
   → 다이어그램 분석
   → AKS와의 연결 이해

3️⃣ Architecture.md 분석 (구조)
   → 시스템 아키텍처 이해
   → 컴포넌트 간 관계 파악

4️⃣ Hands-on.md 직접 실행 (실습)
   → 모든 명령어 직접 입력
   → 예상되는 결과 확인
   → 실패하면 Troubleshooting 참고

5️⃣ PacketFlow.md 추적 (흐름)
   → 패킷이 어떻게 이동하는지 추적
   → 각 단계에서의 변환 이해

6️⃣ Troubleshooting.md 학습 (디버깅)
   → 흔한 문제 해결 방법
   → 진단 명령어 학습

7️⃣ Interview.md 질문 답변 (검증)
   → 개념을 설명할 수 있는가?
   → 심화 질문에 답할 수 있는가?
```

### ❌ 하지 말아야 할 것

- ❌ 이론만 읽고 실습 건너뛰기
- ❌ 스크린샷만 보고 직접 명령어 안 쳐보기
- ❌ 앞 장을 건너뛰고 나중 장만 보기
- ❌ 한 번에 다 이해하려고 서두르기

- 이론 빠르게 스킵

- **예상 총 시간**: 20-30시간

## 💡 효과적인 학습 팁

### 1️⃣ 한 번에 한 장씩
- 여러 장을 동시에 진행하지 말기
- 한 장을 완전히 이해한 후 다음으로
- 각 장의 선수 지식 확인하기

### 2️⃣ 실습 환경 구성
- Linux VM 또는 WSL2 준비
- 필요한 도구 미리 설치
  ```bash
  sudo apt-get install -y \
    net-tools iproute2 bridge-utils \
    iptables tcpdump curl jq docker.io
  ```

### 3️⃣ 노트 작성
- 각 개념을 자신만의 말로 설명
- ASCII 다이어그램을 손으로 그리기
- 질문 목록 작성 (나중에 답하기)

### 4️⃣ 실습 기록
- 실습 결과 스크린샷 또는 로그 저장
- 예상과 다른 결과가 나온 이유 분석
- Troubleshooting 경험 축적

### 5️⃣ 정기적인 복습
- 1주일 후 같은 장 복습
- 면접 질문으로 개념 검증
- "왜?"라는 질문 계속하기

### 6️⃣ 커뮤니티와 공유
- 학습 과정을 다른 사람에게 설명
- GitHub Issue로 질문 올리기
- 팀과 함께 Hands-on 진행

## 🔗 선수 지식 요약

### 반드시 필요한 것
- ✅ Linux 기본 명령어 (ls, cat, ps, grep 등)
- ✅ 터미널/Shell 사용법
- ✅ 파일 시스템 개념
- ✅ 기본적인 IP/CIDR 이해

### 있으면 좋은 것
- ⭐ Docker 기본 사용 경험
- ⭐ Kubernetes Pod/Service 개념
- ⭐ C 언어 기본 이해 (eBPF용)

### 필요 없는 것
- ❌ Kernel 프로그래밍 경험
- ❌ eBPF 사전 지식
- ❌ Kubernetes 관리자 인증
